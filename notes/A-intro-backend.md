# 01 - Intro to Backend

## 🌐 What I got so far
- Backend is basically the brain of an app, it runs on the server side.
- It takes requests from the client (like browser or mobile app) and sends back responses.
- Handles stuff like logic, database, login systems, and security.

## 🔄 Client vs Server
- Client → asks for something (like a webpage).
- Server → processes that request and gives back data.
- They talk using HTTP.

## 🌍 Internet basics
- DNS is like a phonebook, it converts names (google.com) into IP addresses.
- Hosting is just putting your app online so people can reach it.
- Browser is the client that sends requests and shows the response.

## Why backend matters
Backend development serves as the essential infrastructure of web applications, managing the critical logic, security, and data storage that occur away from the user's view. This architectural foundation relies on the synergy between servers, databases, and APIs to process requests and maintain system integrity. While the frontend handles visual elements, the backend ensures scalability and performance through diverse programming languages and specialised frameworks. Effective deployment further involves leveraging cloud services and automation to transition code into a stable production environment. Ultimately, a sophisticated backend is vital for protecting sensitive information and supporting an application’s growth as user demands increase.
